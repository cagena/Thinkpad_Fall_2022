Homework #2: First C++

This exercise is intended to be a simple programming task that helps you with the somewhat complex job of setting up and using PlatformIO, Arduino libraries, and FreeRTOS.

Write a one-file program for your 38-pin ESP32 microcontroller board which produces square waves simulataneously on two pins:

  • GPIO pin 12 should be named FAST_PIN by a #define or a #const and produce a 500 Hz square wave with 50% duty cycle

  • GPIO pin 13 should be named SLOW_PIN and produce a 5 Hz square wave with 50% duty cycle

The GPIO pins (General Purpose Input-Output) pins are called GIOP pins on the diagram on Canvas. Same thing.

You may wish to begin with the file pleasedontcrash.cpp (the example file for Monday of Week 4) and modify its tasks. You should modify the names of the tasks and definitely modify the comments for the tasks.

You must use the functions pinMode() to set up the pins as digital outputs and digitalWrite() to set the pins high (1) and low (0). Task timing should be done using the FreeRTOS function vTaskDelay(). This is an exercise in using tasks, so analogWrite() should not be used.

If you’d like to try the more accurate timing function vTaskDelayUntil(), it is accepted as an improved substitute for the simpler delay function.

You should use an oscilloscope or logic analyzer to test your program. If one is not available, you can use a voltmeter to verify that you’re getting 50% duty cycle square waves (each digital output should be at half the supply voltage), but you’ll need the more sophisticated tools to measure the frequencies.

Name your file your_last_name.cpp in all lowercase letters with underscores replacing spaces and punctuation in your name. Submit that file to this assignment.
