Homework: Pretend Python

In this assignment, you’ll practice writing a C++ module and working with arrays. You do not need to know any Python to complete this exercise.

Write a C++ module which prints some C arrays as if they were Python lists. Your module must consist of two files, print_array.h and print_array.cpp. Each file should have Doxygen style comments at the top of the file and for each function in the file.

Only a few types of data need to be displayed this way. For each type of data, a list display would look something like [0, 1, 2, 3, 4] where the display begins with a square bracket, then the elements are printed with commas between them, and then a closing square bracket ends the display. No end-of-line character is to be printed as part of printing the array. An empty array (given as an array of zero size) should be printed as [].

  • bool - Each item in the array is shown as T if it is true or F if it is false.

  • uint8_t - Each item in the array is shown as an unsigned 8-bit decimal integer.

  • int8_t - Each item in the array is shown as a signed 8-bit decimal integer.

  • float - Each item is a floating-point number, printed as Serial.print (number); would print it in C++. For convenience, it’s nice to set the number of digits after the decimal point to 3, but this isn’t required.

You are to write four overloaded functions named print_array(), meaning they have the same name but accept different types of parameters. The parameters are:

  • A pointer to an array of the given data type

  • The number of elements to be printed

  • A reference to the device on which things are to be printed, with the default value being the device called Serial in Arduino

Each function will return nothing. For example:

void print_array (float* p_array, uint8_t size, Print& device = Serial);

in print_array.h and

void print_array (float* p_array, uint8_t size, Print& device)
{
    // ... your code
}

in print_array.cpp.

You should write a file main.cpp and test your functions with a variety of inputs. The grader will use his own test file, so you do not need to submit your main file.

Place your files print_array.h and print_array.cpp into a ZIP format compressed archive named your_last_name.zip (all lowercase, underscores replacing non-alphanumeric characters) and submit that archive to this assignment.
